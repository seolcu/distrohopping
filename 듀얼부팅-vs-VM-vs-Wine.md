# 듀얼부팅 vs VM vs Wine

리눅스에서 Windows 앱을 실행하는 방법 비교.

## 듀얼부팅

### 장점

- 모든 앱 100% 호환
- 성능 최고

### 단점

- 리눅스와의 공유환경 구성 어려움
- 기기에 따라 부트로더 세팅이 번거로움
- 블루투스 기기 설정 번거로움
- 여기저기 로그인 많이 하게됨

## 가상머신

### 장점

- 리눅스 내에서 아주 간단하게 만들 수 있음
- 복제 가능
- 공유 폴더 존재

### 단점

- 성능 및 메모리 손실
- 가상머신에서의 실행을 차단한 앱 존재(안티치트 있는 게임들)
- 4K 모니터 지원 X
- GPU Passthrough 하려면 일단 GPU가 두개여야함

## Wine (Proton, Bottles 등)

### 장점

- 리눅스와 가장 통합된 형태로 만들 수 있음
- 매우 간편함

### 단점

- 호환 안 되는 경우 많음(Ex. 한컴오피스)
- Wine에서의 실행을 차단한 앱 존재(안티치트 있는 게임들)
- Wine은 에뮬레이터가 아니기에 Bottles를 쓰는 게 아니라면 잘못 사용했을 시 시스템이 박살날 위험이 있음

## 가상머신 + Wine (혼합)

### 장점

- 가상머신과 Wine의 장점을 모두 가짐
- 성능 및 메모리 손실을 Wine으로 커버
- Wine으로 힘든 앱들을 가상머신으로 커버

### 단점

- 안티치트 문제는 여전히 해결못함
